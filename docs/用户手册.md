# Nexus 用户手册

## 什么是 Nexus？

Nexus 是一个本地 Web 控制台，让你可以在单个浏览器界面中同时管理多个 CLI AI Agent 实例（Claude Code、OpenCode 等）的并行协作。

想象一下：过去你需要打开多个终端窗口分别运行不同的 AI 编程助手，而现在只需一个浏览器标签页就能同时监控和操作所有 Agent，还能实时查看文件变化和 Git 状态。

---

## 快速开始

### 安装

```bash
# 安装 Node.js 依赖
pnpm install

# 开发模式运行（推荐）
pnpm dev

# 或者完整开发模式（前后端热重载）
pnpm dev:full

# 生产模式
pnpm build
pnpm start
```

首次运行后，Nexus 会自动在浏览器中打开 http://localhost:7700

### 启动命令

```bash
# 在当前目录启动
nexus

# 指定项目路径启动
nexus ~/projects/my-app

# 初始化项目配置
nexus init

# 查看工作空间状态
nexus status

# 停止服务
nexus stop

# 自定义端口
NEXUS_PORT=8080 nexus start
```

## 界面概览

Nexus 采用四栏布局设计：

```
┌────┬─────────────────────┬────────────┬──────────┐
│ 侧  │    Agent 面板区     │  Diff/评论  │  文件树  │
│ 边  │   (主交互区域)      │            │          │
│ 栏  │                    │            │          │
└────┴─────────────────────┴────────────┴──────────┘
```

### 1. 侧边栏（Sidebar）

左侧 48px 宽的工具栏，包含以下功能图标：

| 图标 | 功能 |
|------|------|
| `+` | 添加新的 Agent 面板 |
| `⚡` | 任务分发（批量创建任务/广播消息） |
| `📋` | 任务模板 |
| `⚙` | 工作空间设置 |

### 2. Agent 面板区（主区域）

每个 Agent 运行在一个可折叠的手风琴面板中，显示：

- **状态指示器**：运行中（绿）、等待中（黄）、空闲（灰）、错误（红）
- **Agent 类型**：claudecode、opencode、kimi-cli、qwencode 等
- **任务描述**：你给该 Agent 分发的任务
- **运行时信息**：模型名称、上下文使用率、累计费用（Claude Code 特有）

面板可以展开查看完整的终端交互界面，也可以折叠为卡片形式快速浏览所有 Agent 状态。

### 3. Diff 与评论面板

- **Git Diff**：查看未 staged 和已 staged 的文件变化
- **代码评论**：对特定代码行添加评论，反馈给指定 Agent

### 4. 文件树

实时显示项目文件结构，支持：

- 目录展开/折叠
- 文件点击预览（带语法高亮）
- 文件变化自动检测（chokidar 监听）

## 核心功能

### 管理多个 Agent

**创建新 Agent 面板**：
1. 点击侧边栏 `+` 图标
2. 选择 Agent 类型（claudecode / opencode 等）
3. 设置工作目录（可选）
4. 填写任务描述
5. 选择恢复模式（continue / restart / manual）

**Agent 状态**：
- `running`：正在执行任务
- `waiting`：等待用户输入
- `idle`：空闲状态
- `stopped`：已停止
- `error`：出错

**重启 Agent**：右键点击面板标题 → 选择重启模式

### Git 工作树隔离

Nexus 支持为每个 Agent 创建独立的 Git worktree，实现真正的并行开发：

```yaml
# .nexus/config.yaml 示例
panes:
  - name: "功能 A"
    agent: claudecode
    worktreePath: ../my-app-feature-a
    branch: feature-a
    task: "开发新功能"

  - name: "功能 B"
    agent: claudecode
    worktreePath: ../my-app-feature-b
    branch: feature-b
    task: "开发另一个功能"
```

这样两个 Agent 可以在不同分支上并行工作，互不干扰。

### 实时元数据监控

Nexus 自动解析 Claude Code 的 statusline 输出，显示：

- **Model**：当前使用的模型（如 claude-sonnet-4-5）
- **Context**：上下文使用百分比
- **Cost**：本次会话累计费用（美元）
- **Session ID**：会话标识符

这些信息会实时更新在每个面板的折叠状态中。

### 终端交互

- 每个 Agent 面板内置完整的 xterm.js 终端
- 支持命令输入、输出滚动、鼠标操作
- 底部浮动 Shell（`__shell__`）：随时可用的全局终端，不属于任何 Agent

### 文件浏览

- **文件树**：左侧实时显示项目结构
- **文件预览**：点击文件，在右侧查看代码（Shiki 语法高亮）
- **变化检测**：文件修改、创建、删除会自动更新树结构

### Git 集成

- **Diff 面板**：查看未 staged 和已 staged 的变化
- **快捷操作**：stage、unstage、discard、commit、push
- **分支信息**：显示当前分支、与远程的 ahead/behind 数量

## 配置文件

### 全局配置 `~/.nexus/config.yaml`

定义系统级设置和 Agent 定义：

```yaml
version: "1"

defaults:
  shell: /bin/zsh
  scrollback_lines: 5000
  grid_columns: 2
  theme: dark-ide

agents:
  claudecode:
    bin: claude
    continue_flag: "--continue"
    statusline: true
    env:
      ANTHROPIC_API_KEY: "${ANTHROPIC_API_KEY}"
```

### 项目配置 `.nexus/config.yaml`

定义当前项目的工作空间：

```yaml
version: "1"
name: "my-app"
description: "项目描述"

repository:
  path: "."
  git: true

panes:
  - id: pane-1
    name: "重构任务"
    agent: claudecode
    workdir: src/auth
    task: "重构 JWT 验证逻辑"
    restore: continue
```

### 运行时状态 `.nexus/agents.yaml`

Nexus 自动生成和维护，记录所有 Agent 的实时状态：

```yaml
updated_at: "2026-03-16T10:00:00+08:00"

panes:
  - id: pane-1
    name: "重构任务"
    agent: claudecode
    pid: 12345
    status: running
    model: claude-sonnet-4-5
    context_used_pct: 35
    cost_usd: 0.15
    session_id: "abc123"
```

此文件会被 gitignore 忽略，用于 Agent 之间互相感知彼此状态。

## 常见场景

### 场景 1：多人 Code Review

1. 创建一个 Agent 面板专门用于 Review
2. 使用 Git Diff 面板查看代码变化
3. 在 Diff 中添加评论
4. 评论会自动发送给指定的 Agent 进行处理

### 场景 2：并行开发多个功能

1. 为每个功能创建独立的 Agent 面板
2. 为每个 Agent 配置独立的 Git worktree 和分支
3. 同时启动多个 Agent 并行开发
4. 通过 `.nexus/agents.yaml` 互相感知进度

### 场景 3：任务分发与广播

1. 创建广播组，将相关 Agent 归为一组
2. 使用任务分发功能批量创建任务
3. 可以向整个组发送广播消息

## 故障排查

### Agent 无法启动

1. 检查 Agent CLI 是否已安装：`which claude` / `which opencode`
2. 检查全局配置 `~/.nexus/config.yaml` 中 Agent 路径是否正确
3. 查看终端输出中的错误信息

### 终端无输出

1. 检查终端是否正常连接（刷新页面重试）
2. 调整终端大小（有些 Agent 对终端尺寸敏感）

### 文件树不更新

1. 检查项目目录是否有读写权限
2. 确认未被 gitignore 的文件才会显示

## 技术架构

```
浏览器 (React + WebSocket)
    │
    ▼
Node.js Server (Fastify)
    │
    ├── PTY Manager (node-pty) ──► CLI Agent 进程
    ├── File Watcher (chokidar) ──► 文件系统
    ├── Git Service (simple-git) ──► Git 仓库
    └── History Manager ──► 终端历史存储
```
